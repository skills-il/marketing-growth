# ערכת כלים לקידום אתרים בעברית

## הוראות

### שלב 1: ניתוח מורפולוגיה של מילות מפתח בעברית
עברית היא שפה שורשית שבה תחיליות משנות משמעות. זהו גרסאות של מילות מפתח:

| תחילית | עברית | משמעות | דוגמה |
|---------|--------|---------|---------|
| ה- | ha- | ה' הידיעה | הבית (the house) |
| ו- | ve- | וו החיבור | והבית (and the house) |
| ב- | be- | ב' השימוש | בבית (in the house) |
| ל- | le- | ל' היעד | לבית (to the house) |
| מ- | me- | מ' המקור | מהבית (from the house) |
| ש- | she- | ש' הזיקה | שבבית (that in the house) |

לכל מילת מפתח מטרה:
1. חלצו את השורש באמצעות ניתוח מורפולוגי
2. צרו את כל צירופי התחיליות שמשתמשים עשויים לחפש
3. כללו צורות סמיכות: "בית קפה" לעומת "הבית של הקפה"
4. התחשבו בצורות זכר/נקבה ויחיד/רבים
5. הריצו `scripts/analyze_keywords.py` על רשימת מילות המפתח לקבלת ספירת גרסאות

### שלב 2: הגדרת SEO לדומיין .co.il
לדומיינים ישראליים יש דרישות אופטימיזציה ספציפיות:

| הגדרה | ערך | הערות |
|--------|------|--------|
| עדיפות TLD | co.il. | מועדף לעסקים ישראליים |
| מיקום שרת | ישראל או CDN קרוב | משפר דירוג מקומי |
| Google Search Console | נכס google.co.il | רישום נפרד מ-.com |
| מפת אתר | כלול הערות hreflang | נדרש לאתרים דו-לשוניים |
| robots.txt | אפשר Googlebot | הגדרה סטנדרטית |

רשימת בדיקה:
1. רשמו דומיין .co.il אצל רשם מוסמך של ISOC-IL
2. הגדירו Google Search Console לנכס ה-.co.il באופן ספציפי
3. הגדירו DNS עם נקודות קצה CDN ישראליות או קרובות
4. ודאו שהאתר נטען נכון עם פריסת RTL (ימין לשמאל)
5. שלחו מפת אתר XML עם הערות hreflang ל-Google Search Console

### שלב 3: הטמעת תגי hreflang
הגדירו טירגוט שפה נכון לתוכן עברי-ישראלי:

```html
<!-- בתוך head של HTML -->
<link rel="alternate" hreflang="he-IL" href="https://example.co.il/page" />
<link rel="alternate" hreflang="en" href="https://example.co.il/en/page" />
<link rel="alternate" hreflang="x-default" href="https://example.co.il/en/page" />
```

למפת אתר XML עם hreflang:
```xml
<url>
  <loc>https://example.co.il/page</loc>
  <xhtml:link rel="alternate" hreflang="he-IL" href="https://example.co.il/page"/>
  <xhtml:link rel="alternate" hreflang="en" href="https://example.co.il/en/page"/>
</url>
```

כללים לאתרים ישראליים:
1. השתמשו תמיד ב-`he-IL` (לא רק `he`) לתוכן עברי ישראלי
2. הגדירו `x-default` לגרסה האנגלית למבקרים בינלאומיים
3. כל דף בשפה אחת חייב לקשר למקבילו בכל השפות האחרות
4. hreflang חייב להיות דו-כיווני (דף עברי מקשר לאנגלי, אנגלי מקשר לעברי)
5. השתמשו בכתובות URL מוחלטות ועקביות בכל הצהרות ה-hreflang

### שלב 4: בניית נתונים מובנים של Schema.org בעברית
צרו סימון JSON-LD מותאם לעסקים ישראליים:

```json
{
  "@context": "https://schema.org",
  "@type": "LocalBusiness",
  "name": "שם העסק",
  "address": {
    "@type": "PostalAddress",
    "streetAddress": "רחוב דוגמה 42",
    "addressLocality": "תל אביב-יפו",
    "addressRegion": "מחוז תל אביב",
    "postalCode": "6100000",
    "addressCountry": "IL"
  },
  "telephone": "+972-3-XXX-XXXX",
  "openingHoursSpecification": [
    {
      "@type": "OpeningHoursSpecification",
      "dayOfWeek": ["Sunday","Monday","Tuesday","Wednesday","Thursday"],
      "opens": "09:00",
      "closes": "18:00"
    },
    {
      "@type": "OpeningHoursSpecification",
      "dayOfWeek": "Friday",
      "opens": "09:00",
      "closes": "14:00"
    }
  ]
}
```

שיקולי Schema ספציפיים לישראל:
1. **שעות שבת:** יום שישי נסגר מוקדם (בדרך כלל 14:00), שבת סגור. השתמשו ב-`OpeningHoursSpecification` ללא רשומת שבת.
2. **תעודת כשרות:** השתמשו ב-`additionalProperty` עם `name: "Kosher Certification"` ופרטי הגוף המכשיר (רבנות, בד"ץ וכו')
3. **פורמט טלפון:** השתמשו תמיד בקידומת בינלאומית `972+`
4. **פורמט כתובת:** השתמשו בתעתיק עברי לשמות רחובות; כללו מיקוד
5. **מטבע:** הגדירו `priceCurrency` ל-`ILS` לכל התמחור

### שלב 5: אופטימיזציה לגורמי דירוג של Google.co.il
שיקולי דירוג ספציפיים לישראל:

| גורם | עדיפות | פעולה |
|-------|---------|--------|
| איכות תוכן עברי | גבוהה | עברית טבעית, לא תרגום מכונה |
| פריסת RTL | גבוהה | dir="rtl" נכון על אלמנט html |
| אופטימיזציה למובייל | גבוהה | 70%+ מהתעבורה הישראלית היא מובייל |
| מהירות דף | גבוהה | שימוש בנקודות CDN ישראליות (Cloudflare TLV) |
| ציטוטים מקומיים | בינונית | רישום ב-Zap, duns.co.il, b144.co.il |
| Google Business Profile | גבוהה | תביעת בעלות ואופטימיזציה עם תיאור בעברית |
| תעודת SSL | גבוהה | נדרשת לכל אתרי .co.il |

שלבי הטמעה:
1. ודאו שכל התוכן העברי נכתב על ידי דוברי שפת אם או נבדק יסודית
2. הגדירו `<html lang="he" dir="rtl">` בכל הדפים העבריים
3. מטבו Core Web Vitals תוך התחשבות ברשתות סלולריות ישראליות
4. צרו ואמתו Google Business Profile עם תיאור עסקי בעברית
5. בנו ציטוטים מקומיים בספריות עסקיות ישראליות (Zap, duns.co.il, b144.co.il)
6. ודאו שלכל התמונות יש טקסט חלופי (alt) בעברית

### שלב 6: יצירת סימון Schema לעסקים ישראליים
לעסקים הדורשים סימון ישראלי מיוחד:

דוגמה לשעות פתיחה מותאמות שבת:
```json
{
  "openingHoursSpecification": [
    {
      "@type": "OpeningHoursSpecification",
      "dayOfWeek": ["Sunday","Monday","Tuesday","Wednesday","Thursday"],
      "opens": "08:00",
      "closes": "20:00"
    },
    {
      "@type": "OpeningHoursSpecification",
      "dayOfWeek": "Friday",
      "opens": "08:00",
      "closes": "14:00",
      "description": "ערב שבת - סגירה מוקדמת"
    }
  ]
}
```

סימון תעודת כשרות:
```json
{
  "additionalProperty": [
    {
      "@type": "PropertyValue",
      "name": "תעודת כשרות",
      "value": "רבנות ירושלים"
    },
    {
      "@type": "PropertyValue",
      "name": "רמת כשרות",
      "value": "מהדרין"
    }
  ]
}
```

### שלב 7: אימות וביקורת
הריצו ביקורת SEO מקיפה:
1. אמתו את כל ה-JSON-LD עם Google Rich Results Test
2. בדקו הטמעת hreflang עם כלי בדיקת תגי hreflang
3. ודאו שתוכן עברי מוצג נכון ב-RTL
4. בדקו תגובתיות למובייל במכשירים נפוצים בישראל
5. הריצו `scripts/analyze_keywords.py --audit` לבדיקת כיסוי מילות מפתח
6. ודאו ש-Google Search Console לא מציג שגיאות hreflang
7. אשרו שכל הרישומים בספריות מקומיות עקביים (NAP: שם, כתובת, טלפון)

## דוגמאות

### דוגמה 1: מחקר מילות מפתח בעברית
המשתמש אומר: "אני צריך מילות מפתח לאתר נדל"ן ישראלי"
פעולות:
1. זיהוי מילות מפתח שורשיות: נדל"ן, דירה, בית
2. יצירת גרסאות מורפולוגיות: הנדל"ן, בנדל"ן, דירות (רבים), בתים (רבים)
3. הכללת צורות סמיכות: סוכן נדל"ן, מחירי דירות
4. הרצת `scripts/analyze_keywords.py --keywords "נדלן,דירה,בית"` לניתוח גרסאות מלא
5. מיפוי גרסאות לדפים לפי עדיפות נפח חיפוש
תוצאה: מפת מילות מפתח מלאה בעברית עם גרסאות מורפולוגיות וצירופי תחיליות

### דוגמה 2: הגדרת hreflang לאתר דו-לשוני
המשתמש אומר: "הגדר hreflang לאתר ה-.co.il שלי בעברית ואנגלית"
פעולות:
1. ביקורת דפים קיימים לזוגות שפה
2. יצירת תגי hreflang link עבור ה-head של HTML בכל דף
3. יצירת מפת אתר XML עם הערות hreflang
4. אימות קישור דו-כיווני בין גרסאות שפה
תוצאה: הטמעת hreflang מלאה עם טירגוט he-IL ו-en

### דוגמה 3: Schema למסעדה ישראלית
המשתמש אומר: "צור נתונים מובנים למסעדה הכשרה שלי בירושלים"
פעולות:
1. בניית JSON-LD של LocalBusiness/Restaurant עם פורמט כתובת ישראלי
2. הוספת שעות פתיחה מותאמות שבת (סגירה מוקדמת ביום שישי, שבת סגור)
3. הכללת תעודת כשרות עם פרטי רבנות
4. הוספת Schema תפריט בעברית עם תמחור בש"ח
תוצאה: סימון JSON-LD מלא מוכן ל-Google Rich Results

### דוגמה 4: ביקורת SEO ל-Google.co.il
המשתמש אומר: "בצע ביקורת לאתר המסחר האלקטרוני הישראלי שלי עבור SEO מקומי"
פעולות:
1. בדיקת איכות תוכן עברי ותצוגת RTL
2. אימות הגדרת דומיין .co.il ו-Google Search Console
3. ביקורת תגי hreflang וטירגוט שפה
4. סקירת Google Business Profile וציטוטים מקומיים
5. הרצת ניתוח מילות מפתח למונחי קטגוריות מוצרים בעברית

## פתרון בעיות

### שגיאה: "hreflang mismatch detected"
סיבה: תגי hreflang אינם דו-כיווניים (דף A מקשר ל-B אבל B לא מקשר חזרה ל-A)
פתרון: ודאו שכל הצהרת hreflang היא הדדית. גם הגרסה העברית וגם האנגלית חייבות להפנות אחת לשנייה.

### שגיאה: "RTL rendering issues"
סיבה: חסר מאפיין dir="rtl" או קונפליקטים ב-CSS עם ברירות מחדל של LTR
פתרון: הגדירו dir="rtl" על אלמנט ה-html בדפים עבריים. השתמשו ב-CSS logical properties (margin-inline-start במקום margin-left).

### שגיאה: "Schema validation failed"
סיבה: סימון JSON-LD מכיל שגיאות או חסרים מאפיינים נדרשים
פתרון: בדקו עם Google Rich Results Test. בעיות ישראליות נפוצות: פורמט טלפון שגוי (חובה להשתמש ב-972+), חסר addressCountry: IL, או קוד מטבע ILS.

### שגיאה: "Keywords not ranking on google.co.il"
סיבה: תוכן עשוי להיות מתורגם מכונה או חסרות גרסאות מורפולוגיות של מילות מפתח
פתרון: ודאו שתוכן עברי כולל צירופי תחיליות טבעיים. השתמשו ב-`scripts/analyze_keywords.py` לזיהוי גרסאות חסרות.
